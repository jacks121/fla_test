# 花苗流程自动化 POC 开发计划（Mock 版）

日期：2026-01-31

## 目标
做一个可演示的 POC，使用 mock 数据和 mock 二维码，证明“扫码即记录”的核心流程可跑通。

## POC 边界
### 包含
- mock 标签（二维码/短码）
- mock 数据（花苗/培养皿/位置/人员）
- 核心流程：创建/拆分/合并、批量上架、状态更新、换皿转移
- 事件记录与基础查询
- 单页面手机 Web 应用（浏览器访问）

### 不包含
- 真实打印/真实扫码枪
- 实际库存管理
- 复杂权限、审计、报表

## 关键假设（POC）
- 扫码可用：用 mock QR 图片或输入短码模拟
- 数据可用：内置样例数据或本地轻量存储
- 性能不作为验收重点

---

## 迭代计划（POC）

### Iteration 0：POC 骨架（1-2 天）
**目标**：可启动的演示壳
- 基本数据结构（plant/dish/event/location/staff）
- mock 数据种子（10-50 条）
- 基础 UI 框架（手机优先、单页面）
- 单页面导航结构（模式切换/分区导航）

**验收**：进入页面能看到 mock 列表与演示入口

### Iteration 1：核心扫码流程（2-4 天）
**目标**：演示“扫码即记录”
- 创建/拆分/合并：父一次 + 多子连扫
- 批量上架：位置一次 + 多皿连扫
- 事件记录落库（本地或内存）
- 顺序校验（父→子/位→皿）

**验收**：完成一次完整“拆分 + 上架”的演示

### Iteration 2：状态与转移（1-2 天）
**目标**：补齐关键流程
- 状态更新（感染/变异/正常等）
- 换皿/转移（旧→新）
- 最近 1-3 步撤销

**验收**：可演示转移与状态更新

### Iteration 3：展示与复盘（1 天）
**目标**：形成可讲解的 demo 路径
- 事件列表与过滤（按人/时间/类型）
- Demo 脚本（步骤说明）
- 反馈记录表

**验收**：可对外演示并记录问题

---

## 需求拆分（POC Backlog）

### Epic A：Mock 标签与扫码模拟
- A1 生成 mock 二维码与短码（静态图片/预置）
- A2 扫码替代方式：上传图片或输入短码
- A3 扫码错误提示

### Epic B：事件模型与数据
- B1 事件类型定义（create/split/merge/place/status/transfer）
- B2 plant_id 与 dish_id 分离
- B3 事件记录保存（本地/内存）
- B4 事件查看与回放（列表）

### Epic C：核心流程 UI
- C1 创建/拆分/合并（连扫模式）
- C2 批量上架（位置连扫）
- C3 状态更新
- C4 换皿/转移
- C5 最近撤销

### Epic E：单页面导航与结构
- E1 单页面信息架构（底部 Tab：拆分/上架/状态/转移）
- E2 当前操作上下文提示（减少误扫）
- E3 移动端可访问性与布局适配

### Epic D：演示与反馈
- D1 Demo 流程脚本
- D2 反馈记录与问题清单

---

## POC 验收标准
- 不依赖真实硬件，流程可完整演示
- 拆分/上架/状态/转移四个流程可跑通
- 事件记录可查询
- 演示步骤清晰、可复现
- 手机浏览器单页面可访问、可完成完整 demo
